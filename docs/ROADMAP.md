# Roadmap

Planejamento macro do projeto, por fases. O detalhamento de cada item vive nas Issues (ver `issues.csv` e convenção `[Fase][Categoria] Título`).

- [ ] **F0 — Setup**: configuração inicial do repositório, ferramentas e ambiente
- [ ] **F1 — Core**: regras e lógica principal do domínio
- [ ] **F2 — Network**: comunicação, API ou sincronização (se aplicável)
- [ ] **F3 — UI**: interface e experiência do usuário
- [ ] **F4 — Mobile**: adaptação/publicação mobile (se aplicável)
- [ ] **F5 — Release**: testes finais, documentação e publicação

> Sem detalhamento excessivo aqui — cada fase vira Issues específicas conforme o desenvolvimento avança.
